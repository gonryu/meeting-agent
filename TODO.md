## [x] 녹음 파일 업로드 → 화자 분리 회의록 생성

**현황**
Google Meet 트랜스크립트(Google Docs)만 지원. 오프라인/대면 미팅은 수동 노트 세션만 가능.

**목표**
Slack DM에 녹음 파일(MP3, MP4, M4A, WAV 등)을 업로드하면 화자를 구분하여 회의록 자동 생성.

**구현 고려사항**
- Slack `file_share` 이벤트에서 audio/* MIME 타입 감지 (명함 이미지와 동일 진입점)
- STT + 화자 분리(Diarization) 방법 검토:
  - **Google Cloud Speech-to-Text v2** — 화자 분리(`diarizationConfig`) 내장, 한국어 지원
  - **AssemblyAI** — 화자 분리 + 한국어 지원, REST API 단순
  - **Whisper (OpenAI)** — 로컬 실행 가능, 화자 분리는 pyannote.audio 별도 결합 필요
- 트랜스크립트 형식: `[화자1] 내용\n[화자2] 내용` → 기존 `minutes_internal_prompt`에 그대로 전달 가능
- 화자 이름 매핑: STT가 `SPEAKER_01`, `SPEAKER_02` 등으로 반환하면 참석자 목록과 매핑하는 단계 필요
- 파일 크기 제한: Slack 파일 다운로드 후 STT API 업로드 (대용량 파일 처리 고려)
- 최근 캘린더 이벤트와 자동 매칭하여 미팅 제목·참석자 정보 활용

## [x] 오래 걸리는 커맨드에 진행 상황 중간 메시지 표시

**현황**
브리핑, 기업/인물 리서치, 회의록 생성 등 일부 커맨드는 처리 시간이 수십 초 이상 걸리지만 완료 전까지 아무런 피드백이 없음.

**목표**
처리 단계마다 중간 진행 메시지를 Slack으로 발송하여 사용자가 진행 중임을 인지할 수 있도록 함.

**구현 고려사항**
- 진행 상황을 알려야 할 주요 커맨드:
  - 브리핑 (`/브리핑`, 자동 브리핑): 업체 리서치 중 → 인물 리서치 중 → 브리핑 생성 중
  - 기업/인물 리서치 (`/기업`, `/인물`): ParaScope 조회 중 → Gmail 검색 중 → 웹 검색 중
  - 미팅 생성 (`/미팅추가`): 이메일 조회 중 → 캘린더 등록 중
  - 회의록 생성 (`/미팅종료`): 트랜스크립트 탐색 중 → 회의록 작성 중
- 구현 방식 검토:
  - **단계 시작 시 메시지 발송**: 각 단계 앞에 `_post(..., text="🔍 업체 리서치 중...")` 추가
  - **메시지 업데이트**: 첫 메시지를 `chat_postMessage`로 발송 후 `chat_update`로 내용 갱신 (단일 메시지 유지)
  - 단계별 발송이 단순하고 스레드 추적이 쉬워 우선 검토
- Slack 3초 응답 제한: 슬래시 커맨드는 `ack()` 즉시 호출 후 백그라운드 스레드에서 진행 메시지 발송 (현재 구조와 동일)
