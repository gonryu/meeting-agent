- **회의 제목**: {{title}}
- **회의 일자**: {{date_str}}
- **사용자 검토 개요(있는 경우 — 사용자가 확인한 톤/방향)**:

## 작업

아래 6개 섹션을 모두 포함한 한국어 제안서 마크다운을 작성합니다 (순서 고정):

```markdown
# {{title}} 제안서

## 개요
- 제출처(고객사·부서)
- 제출일: {{date_str}}
- 한 문단 핵심 메시지 (Win Theme를 녹여서)

## 고객 이해
- 고객의 needs/Pain Point 정리
- 결정 기준·관심사
- (있다면) 의사결정 구조 표

## 제안 솔루션
- 솔루션 개요 + 핵심 가치
- 구성요소 표 (구성요소 / 설명 / 해결하는 needs)
- 수행 방법론 + 단계별 일정 표

## 차별화
- Win Theme 3개 이내, 각 테마에 메시지 + 근거
- USP를 고객 가치 표현으로 정리
- (있다면) 고객 우려에 대한 우리 대응

## 일정·비용
- 단계별 일정 요약
- 가격 옵션 표 (옵션명 / 범위 / 추정 금액 / 적합 상황)
- ⚠️ 가격은 검토용 초안임을 명시 (`draft_for_review` 표기)

## 다음 단계
- 의사결정 요청 사항
- 즉시 실행 항목
- 후속 미팅 제안 (회의록에 명시된 것만)
```

- **가격 표기**: pricing_strategy.draft_for_review가 true이면, 일정·비용 섹션 끝에 다음 문구를 붙이세요:
  > _※ 본 가격은 내부 검토용 초안입니다. 실제 제출 전 확인이 필요합니다._
- **회피 영역**: differentiation.avoid_topics 항목은 본문에서 강조하지 마세요.
- **마크다운만**: 코드펜스·설명·JSON 없이 마크다운 본문만 출력.
