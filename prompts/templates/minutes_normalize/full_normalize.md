  - 제목: {{title}}

- **표준 회의록 템플릿 (참고용)**:
  ```markdown

  ```
- **현재 회의록 원문 (보정 대상)**:


## 작업

위 원문을 표준 7섹션(internal) 또는 10섹션(vendor/external/mixed) 양식에 맞게
**재구조화**합니다. 단, 다음의 보존 규칙을 **절대로** 어기지 마세요.

## 보존 규칙 (위반 시 실패)

1. **사용자가 추가한 모든 콘텐츠를 반드시 보존**합니다.
   - 표준 7/10섹션 외에 사용자가 직접 작성한 섹션 (예: `## 후속 메모`, `## 사후 분석`,
     `## 개인 메모`, `## TODO` 등)은 **그대로 보존**하여 회의록의 **마지막 부분**에 모아 둡니다.
   - 사용자가 표준 섹션 안에 추가로 적은 메모, 한 줄 코멘트도 그 섹션 안에 보존합니다.
2. **숫자·날짜·인용·고유명사**는 한 글자도 바꾸지 않고 **원문 그대로** 유지합니다.
   - 단순 오타 교정조차 금지. 양식만 손봅니다.
   - 통화 단위, 기한, 비율 등은 절대 재계산·재포맷하지 않습니다.
3. **콘텐츠 삭제 금지**.
   - 짧거나 비어있는 항목이라도 절대 삭제하지 않습니다.
   - 중복처럼 보이는 항목도 두 위치 모두 사용자 의도일 수 있으므로 보존합니다.
4. **양식만 보정합니다**:
   - 누락된 표준 섹션 헤더 추가 (`## 회의 개요`, `## 결론`, `## 액션아이템` 등)
   - 빈 섹션은 "해당 없음"으로 채움 (단, 사용자가 작성한 내용이 있으면 그대로 둠)
   - 액션아이템 표 형식 정렬 (이미 정보가 있으면 표로 옮기되 내용은 유지)
   - 기존에 있던 `## 출처 로그` / `## 관련 문서` 섹션은 위치만 끝으로 이동시키고 내용은 보존
5. **위키링크 보정**: 알려진 엔티티 목록에 있는 이름이 본문에 평문으로 나오면 `[[이름]]` 으로 감쌉니다.
   - 이미 `[[]]` 로 감싸진 곳, 코드블록 / frontmatter 내부는 손대지 않습니다.
6. **frontmatter 재구성**:
   - 누락 시 새로 합성. `title`, `date`, `type: meeting`, `meeting_type`,
     `stage`, `status`, `source_refs`, `related_entities`, `related_notes`, `tags` 키 포함.
   - 기존 frontmatter 가 있으면 누락 키만 채우고 사용자 값은 보존합니다.
7. **JSON·코드펜스 래핑 금지**. 최종 보정된 회의록 마크다운만 그대로 출력하세요.

## 출력

보정된 회의록 마크다운 전체를 출력합니다. (frontmatter 포함)
설명·머릿말·꼬릿말 모두 금지. 마크다운 본문만.
