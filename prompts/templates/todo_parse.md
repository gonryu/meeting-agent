사용자가 보낸 메시지에서 Todo(할 일) 정보를 추출해줘. 오늘 날짜는 {{today}}이야 (요일: {{weekday}}).

메시지: "{{text}}"

JSON 형식으로만 답변 (다른 텍스트 없이):
{
  "task": "할 일 본문 (트리거 키워드 제거 후 핵심만)",
  "category": "work | personal | ai",
  "due_date": "YYYY-MM-DD" 또는 null,
  "original_text": "원본 메시지",
  "_is_past_completion": false
}

추출 규칙:
- task: "할 일 추가", "할일 추가", "todo 추가", "todo:", "투두:", "추가해줘" 등 트리거 단어를 제거한 핵심 내용만 남김
  - 해시태그(#업무, #개인, #AI)·날짜 표현·"~까지", "~안에" 도 제거
  - 빈 본문이면 task에 빈 문자열을 넣어도 되지만 가능한 한 의미 있는 본문 유지
  - 따옴표·괄호로 감싸진 경우 안쪽 텍스트만 사용 가능
- category 결정 우선순위:
  1) 해시태그 명시: `#업무` 또는 `#work` → "work", `#개인` 또는 `#personal` → "personal", `#AI` 또는 `#ai` → "ai"
  2) 해시태그 없으면 메시지 의미로 추론
     - 회사/회의/제안서/고객/업체 관련 → "work"
     - 병원/약속/취미/가족/장보기 → "personal"
     - LLM/모델/프롬프트/AI 도구/meeting-agent/Claude/ChatGPT 자체 논의 → "ai"
  3) 모호하면 기본값 "work"
- due_date 자연어 처리 (KST 기준, 오늘 = {{today}}, 요일 = {{weekday}}):
  - "오늘" → 오늘 날짜
  - "내일" → 오늘 +1일
  - "모레" → 오늘 +2일
  - "다음주 월요일" / "다음주 금요일" 등 → 다음 주 해당 요일
  - "이번주 금요일" → 이번 주(일~토 기준) 해당 요일
  - "이번달 말" → 이번 달 마지막 일
  - "다음달 초" → 다음 달 1일~5일 사이로 추정 (정확한 날짜는 다음달 1일)
  - "5/3", "5월 3일" → 올해 해당 날짜 (이미 지났으면 내년)
  - "2026-05-03" 등 ISO 형식 그대로
  - "1주일 뒤" → 오늘 +7일
  - 명시 없으면 null
- _is_past_completion: 메시지가 과거형 완료 표현으로 끝나면 true (예: "~했어", "~끝냈어", "~완료", "~다 했어"). 이런 경우 호출자가 "이미 완료된 항목인가요?"라고 다시 묻는다.
  - "~할 거야", "~해야 해", "~예정" 처럼 미래/예정 표현은 false
- original_text: 사용자가 보낸 원문을 그대로 (트리거 키워드 포함)
