# 세션 작업 일지

이 폴더는 *날짜·테마별* 작업 일지를 모아둡니다. 한 번 자리잡힌 패턴은 [CLAUDE.md](../../CLAUDE.md) 본문이나 `docs/`로 승격되지만, 세션 단위의 "오늘 뭘 했고 어디까지 왔는지"는 여기서 추적합니다.

## 파일 명명 규칙

`YYYY-MM-DD-slug.md` — 같은 날 여러 테마면 슬러그로 분리.

예시:
- `2026-05-08-trello-thread-register.md`
- `2026-05-08-perf-debug.md` (만약 같은 날 다른 큰 작업이 있었다면)

## Claude를 위한 지침 — "출근" / "이어서 하자" / "지난번 업무 이어서 하자"

사용자가 다음 표현 중 하나를 *새 세션 첫 메시지로* 말하면:
- "출근"
- "이어서 하자"
- "지난번 업무 이어서 하자"
- "어제 하던 거"
- "어디까지 했지?"

다음 절차로 컨텍스트를 복원합니다:

1. **이 폴더의 가장 최근 파일을 읽는다** — `ls docs/sessions/*.md | sort -r | head -1` 또는 git log로 가장 최근 변경된 세션 일지
2. **현재 git 상태 점검** — `git status`, `git log -10 --oneline`, 작업 브랜치 확인
3. **세션 일지의 *5번 섹션 (다음 세션 후보 작업)* 읽기** — 거기에 미해결 항목들 정리됨
4. **3~5줄로 사용자에게 요약** — "지난번에 X까지 마무리했고, Y가 열려있어요. 어떤 거부터?"
5. **사용자 응답 받고 진행**

요약 형식 예시:
```
지난번(2026-05-08)에 ParaMee의 트렐로 스레드 등록 기능을 완성했어요. PR #30~#40 머지 완료, 매니페스트도 적용됐고 사용자 컨펌까지 받았어요.

열려있는 것:
- 자동 등록 흐름에도 구조화 코멘트 적용
- 새 기능에 대한 단위 테스트
- 사용자 피드백 모니터링

오늘은 어떤 거 작업할까요?
```

## 일지 작성 시점 (Claude가 알아서 만든다)

다음 트리거 중 하나일 때 새 세션 일지를 만들어 이 폴더에 둡니다:
- 사용자가 "오늘 작업했던 이력 정리해줘" / "세션 마무리하자" / "기록 남기자" 류 요청
- 한 세션에서 PR 3개 이상 머지하거나, 큰 기능 1건을 끝낸 시점
- 사용자가 "이거 다음에 또 할 수 있게 정리해" 류 요청

## 일지 작성 형식 (템플릿)

```markdown
# 세션 작업 일지 — YYYY-MM-DD

**테마**: 한 줄 요약
**시작 상태**: main 커밋 해시 + 브랜치
**종료 상태**: main 커밋 해시 + 브랜치

## 1. 무엇을 만들었나
- 핵심 PR 표 (#번호, 종류, 한 줄)
- 곁가지 산출물 (문서·매니페스트 등)

## 2. 디버깅 사이클의 교훈
- 증상 → 원인 → 단서 표
- 범용 원칙 (재사용 가능한 것)

## 3. 사용자 환경 / 운영 상태
- 라이브 서버, 배포, Slack 설정 등

## 4. 마지막 사용자 컨펌
- "이 시점에 사용자가 무엇을 확인했는가"

## 5. 다음 세션 후보 작업 (열린 항목)
- [ ] checkbox로 미해결 항목

## 6. 코드 변경 파일

## 7. 관련 메모
- 사용자 선호·운영 룰 등
```

5번 섹션이 핵심 — 이게 다음 세션의 시작점이 됩니다.
