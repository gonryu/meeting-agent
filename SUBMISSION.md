## 2. 프로젝트 개요

### 해결하려는 문제

외부 미팅이 잦은 영업·사업팀은 미팅마다 상대 업체 조사, 회의록 작성, 액션아이템 관리, 외부 공유용 회의록 발송을 수작업으로 반복해야 한다. 미팅 건수가 많을수록 준비와 사후 처리에 소요되는 시간이 선형적으로 증가하며, 맥락이 누락되거나 후속 조치가 지연되는 문제가 발생한다.

### 솔루션 한 줄 요약

Slack 하나로 미팅 전 브리핑 자동 준비 → 미팅 중 회의록 자동 생성 → 미팅 후 외부 발송·액션아이템 관리까지 미팅 전 주기를 자동화하는 AI 어시스턴트.

## 4. AI 활용 방식 및 주요 프롬프트

### 사용한 AI 도구

| 도구 | 역할 | 모델 |
|------|------|------|
| Google Gemini API | 웹 검색 + 텍스트 생성 (주 LLM) | `gemini-2.0-flash` |
| Anthropic Claude API | Gemini 할당량 초과 시 폴백 LLM | `claude-haiku-4-5` |

Gemini는 `GoogleSearch` 툴을 함께 호출해 실시간 웹 정보를 검색·요약한다. Gemini가 429(할당량 초과) 응답을 반환하면 Claude의 `web_search_20250305` 베타 툴로 자동 폴백된다.

```python
# 주 LLM (Gemini + 웹검색)
_gemini.models.generate_content(
    model="gemini-2.0-flash",
    contents=prompt,
    config=GenerateContentConfig(tools=[Tool(google_search=GoogleSearch())])
)

# 폴백 (Claude + 웹검색)
_claude.beta.messages.create(
    model="claude-haiku-4-5",
    tools=[{"type": "web_search_20250305"}],
    betas=["web-search-2025-03-05"]
)
```

### 기존 방식 대비 달라진 점

| 항목 | 기존 방식 | Meeting Agent |
|------|-----------|---------------|
| 미팅 브리핑 준비 | 담당자가 직접 업체 검색·정리 (30~60분) | 자동 웹 검색 + Drive 캐시로 즉시 발송 |
| 회의록 작성 | 미팅 후 수작업 정리 (30~90분) | Google Meet 트랜스크립트 + 메모 기반 자동 생성 |
| 외부 회의록 공유 | 내부 회의록에서 민감정보 수동 삭제 후 발송 | 내부용 → 외부용 2단계 자동 생성 후 Slack 버튼 1회 클릭으로 발송 |
| 액션아이템 관리 | 이메일/메신저로 개별 전달 | 회의록에서 자동 추출 + Slack DM + 매일 08:00 리마인더 |

## 5. 주요 기능 설명

### ① 미팅 브리핑 자동 발송 (Before Agent)

매일 09:00 KST (또는 수동 `/브리핑` 명령) 에 Google Calendar를 조회해 현재 시각 기준 24시간 이내 미팅을 감지한다. 외부 미팅으로 판단되면 아래 내용을 포함한 브리핑을 Slack DM으로 발송한다:

- **업체 동향**: Gemini 웹 검색 기반 최신 뉴스 (7일 캐시, Drive 저장)
- **담당자 정보**: Gmail 히스토리 + 웹 검색 기반 공개 정보
- **서비스 연결점**: 우리 제품·서비스와 상대 업체의 시너지 포인트
- **이전 미팅 맥락**: Drive에 저장된 이전 회의록 + Gmail 최근 이메일
- **어젠다**: Calendar에 등록된 어젠다 표시, 없으면 스레드 답장으로 등록 유도

자연어로 미팅 생성도 가능하다: `"내일 오후 3시에 KISA 미팅 잡아줘"` → LLM 파싱 → Google Calendar 이벤트 생성 + Google Meet 링크 자동 포함 + 즉시 브리핑 발송.

### ② 회의록 자동 생성 (During Agent)

`/미팅시작` → 메모 입력 → `/미팅종료` 흐름으로 회의를 관리한다. 종료 시점에 Google Meet 트랜스크립트(Drive 자동 저장)를 백그라운드로 폴링하고, 트랜스크립트가 확인되면 메모와 합쳐서 LLM으로 회의록을 자동 생성한다. 90분 이내 트랜스크립트가 없으면 메모만으로 생성한다.

생성된 회의록은 두 버전으로 Drive에 저장된다:

- `{날짜}_{제목}_내부용.md`: 전략적 판단·내부 메모 포함
- `{날짜}_{제목}_외부용.md`: 내부 정보 제거, 공유 가능한 버전

Slack에서 두 Drive 링크를 바로 확인할 수 있다.

### ③ 미팅 후 자동화 (After Agent)

회의록 생성 직후 아래 작업이 자동으로 처리된다:

- **외부 회의록 이메일 발송**: Slack에서 "발송하기" 버튼 클릭 한 번으로 외부 참석자에게 HTML 이메일 발송 (Gmail API)
- **액션아이템 추출 및 알림**: LLM이 회의록에서 담당자·내용·기한을 추출하여 SQLite에 저장, Slack DM으로 담당자에게 알림, 매일 08:00 KST 마감 임박 항목 리마인더 발송
- **Contacts 자동 업데이트**: 외부 참석자의 Drive People 파일에 최근 미팅 날짜 및 이력 자동 기록

## 7. 아쉬운 점 / 개선 여지

**① LLM 할당량 의존성**
Gemini 무료 티어의 분당 요청 한도로 인해 브리핑 시 여러 미팅을 동시에 처리할 경우 Claude 폴백으로 넘어가는 빈도가 높다. Claude는 실시간 웹 검색이 제한적이어서 업체 최신 동향 품질이 떨어진다. Gemini 유료 플랜 전환이 필요하다.

**② 서버 재시작 시 상태 유실 위험**
`_pending_agenda` (어젠다 대기 목록)는 JSON 파일로 영속화하지만, 다중 인스턴스 환경에서는 인스턴스마다 별도 메모리 상태를 가져 충돌이 발생한다. Redis나 DB 기반 상태 공유가 필요하다.

**③ Gmail 이메일 맥락 검색 정확도**
회사명 키워드(`"KISA"`)로 Gmail 전문 검색 시 해당 회사와 무관하지만 본문에 키워드가 포함된 이메일이 함께 검색된다. 발신/수신 도메인 필터를 추가하면 정확도가 크게 향상될 것이다.

**④ Google OAuth 토큰 자동 갱신 미구현**
토큰 만료 시 사용자가 `/재등록`으로 수동 재인증해야 한다. `refresh_token`을 이용한 자동 갱신 로직을 추가해야 한다.

**⑤ 미구현 기능**
Trello 연동(액션아이템 → Trello 카드 자동 생성), 미팅 유형별 어젠다 템플릿, 파일 업로드 처리(명함 OCR, 음성파일 STT), 미팅 10분 전 알림 등은 설계는 됐지만 구현하지 못했다.

## 8. 기대 효과

### 업무 효율화

- 외부 미팅 1건당 준비(30~60분) + 회의록 작성(30~90분) = **총 1~2.5시간 절감**
- 주 10건 미팅 기준 매주 10~25시간의 반복 작업 제거
- 준비 불충분으로 인한 미팅 품질 저하 방지: 업체 동향·담당자 정보·이전 맥락을 미팅 전 자동 확보

### 지식 자산화

- 모든 외부 미팅의 회의록이 Drive에 구조화된 형태로 축적됨
- 새로운 미팅 전 이전 미팅 맥락을 자동 참조 → 업체 관계 연속성 유지
- Contacts 파일에 미팅 이력이 누적되어 CRM 대체 효과

### 협업 품질 향상

- 내부용/외부용 회의록 분리 생성으로 민감정보 노출 위험 제거
- 액션아이템 자동 추출 + Slack 알림으로 후속 조치 누락 방지
- 외부 회의록 이메일 발송 자동화로 파트너사와의 커뮤니케이션 속도 향상
